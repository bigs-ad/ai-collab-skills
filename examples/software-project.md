# Example: Software Project

## Prompt

Use `$start-project`: "I want to build a local tool that turns invoices into a monthly report."

## Expected Route

1. `start-project` creates a brief, scope, constraints, and workstreams.
2. `manage-project` tracks docs, parser, UI, export, and tests.
3. `add-feature` handles each accepted feature.
4. `fix-bug` handles regressions.
5. `check-work` verifies parser accuracy, UI behavior, and export output.

## Notes

Do not require a full enterprise process before a prototype exists. Start with QuickProbe or Candidate, then Formalize after the approach is accepted.
