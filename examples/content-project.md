# Example: Content Project

## Prompt

Use `$start-project`: "I want to create a short course and publish weekly lessons."

## Expected Route

1. `start-project` defines audience, promise, format, cadence, and source material.
2. `manage-project` tracks outline, drafts, review, assets, and publishing.
3. `delegate-work` can split lesson outline, examples, and proofreading.
4. `check-work` reviews completeness, consistency, and publication readiness.

## Notes

Treat editorial review as verification. Do not force software-only artifacts such as tests unless the content project needs them.
