# Add Feature

## Workflow

1. Clarify feature goal, user value, non-goals, and acceptance criteria.
2. Check whether the request changes scope, architecture, business rules, data, platform behavior, or release plans.
3. Choose QuickProbe for direction finding, Candidate for reviewable design, Formalize for approved implementation, or GateReview for high-risk scope.
4. Implement only after scope and acceptance are clear.
5. Verify and record documentation or test updates needed.

## Output

Use `assets/templates/feature-brief.md`.

Do not begin coding when approval is missing for changes to scope, business rules, monetization, release plan, core product rules, architecture, data contracts, platform behavior, or acceptance criteria. Use GateReview.
